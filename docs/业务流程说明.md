# 业务流程图说明

## 核心挂号流程
1. **浏览与查询**
   - 患者进入首页，查看\`Department\`列表。
   - 选择科室，查询该科室下的\`Doctor\`。
   - 选中医生，查看未来7天的排班情况 (\`Schedule\`)，状态包括：可预约、约满、停诊。

2. **预约流程**
   - 选中某个可预约的时段，系统提示选择就诊人（若无，则导向添加表单）。
   - 提交预约请求至后端 \`POST /api/v1/appointments\`。
   - 后端逻辑：
     - 校验当前身份与登录状态。
     - 验证排班状态与可用号源 (\`available_slots > 0\`)。
     - 验证业务规则：同一个就诊人(\`patient_id\`)+同科室+同一天只能预约1次。
     - 数据库事务开启：扣减 \`Schedule.available_slots\` -1 （行级锁或乐观锁防超卖）。
     - 创建 \`Appointment\` 记录，生成唯一预约编号。
     - 提交事务。
   - 前端接收成功结果，跳转“预约成功”详情页。

3. **取消规则**
   - 患者在“我的预约”列表点击取消。
   - 后端验证当前时间距离预约创建时间 \`created_at\` 是否 < 30分钟。
   - 允许取消则退回号源 (\`available_slots + 1\`)，更改状态为 \`CANCELLED\`。
