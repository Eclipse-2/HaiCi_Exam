# 数据库设计

## 核心数据表 (MySQL)

### 1. User表 (患者/管理员用户)
- \`id\` (PK)
- \`phone\` (或 email, 唯一)
- \`password_hash\`
- \`role\` (USER, ADMIN)
- \`created_at\`

### 2. Patient表 (就诊人信息)
- \`id\` (PK)
- \`user_id\` (FK -> user.id)
- \`name\`
- \`id_card\` (身份证号, 唯一)
- \`phone\`

### 3. Department表 (科室)
- \`id\` (PK)
- \`name\` (内科、外科等)
- \`description\`

### 4. Doctor表 (医生)
- \`id\` (PK)
- \`dept_id\` (FK -> department.id)
- \`name\`
- \`title\` (主任医师, 副主任医师等)
- \`specialty\` (专长)
- \`description\`

### 5. Schedule表 (医生排班与号源)
- \`id\` (PK)
- \`doctor_id\` (FK -> doctor.id)
- \`date\` (日期)
- \`session\` (MORNING, AFTERNOON)
- \`total_slots\` (总号源)
- \`available_slots\` (剩余可用号源)
- \`status\` (AVAILABLE, FULL, SUSPENDED)

### 6. Appointment表 (预约记录)
- \`id\` (PK)
- \`appointment_no\` (唯一预约号)
- \`user_id\` (FK -> user.id)
- \`patient_id\` (FK -> patient.id)
- \`schedule_id\` (FK -> schedule.id)
- \`status\` (PENDING, FINISHED, CANCELLED)
- \`created_at\` (记录创建时间，用于判断30分钟免费取消)
